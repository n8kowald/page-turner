## Page Turner 4.0.1

Patch release fixing several bugs — including settings that had been silently broken since the Manifest V3 migration — plus sharper on-page arrows and some security hardening.

### Bug fixes
- **Settings now save again** — the popup's Show/Hide arrows and Prerender toggles silently stopped persisting when the extension moved to Manifest V3 (`chrome.tabs.executeScript` was removed and threw before the save ran). Settings now save correctly *and* apply to open tabs instantly.
- **Prerendering works again** — Chrome removed the old `<link rel="prerender">` mechanism years ago, so the "Prerender next page" option did nothing. It now uses the modern Speculation Rules API for instant page turns.
- **Arrow click feedback fixed** — the pressed-state flash on the on-page arrows only worked once per page; it now fires on every arrow-key press.
- **Stale toolbar icons** — fixed a case where a newly opened tab could briefly inherit the icon state of a previously closed tab.

### Improvements
- **Vector page arrows** — the on-page `‹` `›` indicators are now SVG: crisp at any zoom level and on high-DPI displays, with a subtler look.
- **Smaller attack surface** — the extension no longer exposes any web-accessible resources, so websites can no longer probe for its presence; the background script also validates all icon messages against an allowlist.
- Removed dead code left over from the Manifest V2 era (`content_script.js`).

### Under the hood
No new permissions requested — this release *removes* the `web_accessible_resources` manifest entry.
