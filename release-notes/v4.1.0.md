# Page Turner 4.1.0

Minor release: Page Turner now detects a lot more pagination it used to miss, and the popup has been rebuilt around toggle switches. Permissions are unchanged.

## Bug fixes

**Prerendering no longer hijacks the toolbar icon**
When Page Turner asked the browser to preload the next page, that hidden page could report its own back/next state and overwrite the icon on the page you were actually looking at. The prerendered document's icon is now held back until the page is genuinely shown, and the background script ignores messages from non-active documents.

**Links that require JavaScript no longer swallow a page turn**
`javascript:` hrefs, empty links, and bare `#` links were sometimes picked as the "next" target, so pressing the arrow key did nothing (and on sites with a strict CSP, such as XenForo forums, could be blocked outright).
Only real navigable links (`http`, `https`, `file`) are accepted now.

## Improvements

**Smarter pagination detection** — Page Turner now understands several cases it previously walked past:

- **`rel="next"` / `rel="prev"` links** are recognised (case-insensitively) and preferred over guessing from link text — the unambiguous, standards-based signal wins.
- **Button- and form-based pagination** is found more reliably: likely pagination regions (`nav`, `.pagination`, `.pager`, `[role="navigation"]`, …) are scanned first, and when a form has several buttons the one whose label matches the direction is submitted. This fixes engines like Startpage that paginate via a form's aria-labelled button.
- **Multi-line link text** (a label split across lines) now matches.
- **Fewer false positives** — oddly-cased words like "NeXT" are treated as brand names, not a "next" button, so they no longer trigger a stray arrow.

**Redesigned popup** — the settings popup now uses modern on/off toggle switches instead of radio buttons, with a cleaner header and iconography. It's also more accessible (`aria-pressed`, `aria-label`, proper `lang`/`charset`).

**Honest prerender setting** — on browsers without the Speculation Rules API (Brave, Firefox), the "Instant page turns" toggle now greys out with a short explanation instead of pretending to work.

## Under the hood

- Preferences are read once at startup and cached (one storage round-trip instead of several), and the cache stays in sync with live setting changes.
- Removed the `first_run` global and other Manifest V2-era leftovers; added thorough JSDoc throughout `page-turner.js`.
- Refreshed the icon set (16/32/48/128 px), dropped unused 21/96 px icons, and removed heavyweight `.psd` artwork sources from the repo.
- Added 10 QUnit tests covering the new `rel`, container, submitter, casing, and `javascript:`-link cases.
- No new permissions requested — the manifest change is limited to the version, icon set, and store description wording.
