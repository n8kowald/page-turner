# Page Turner 4.2.0

Minor release: two more real-world pagination layouts are now detected, so the arrow keys work on sites where they previously did nothing. Permissions are unchanged.

## Improvements

**Wider pagination detection** — Page Turner now recognises two more link styles it used to walk straight past:

- **Icon-only arrows labelled by `aria-label`** — pagination links that are just an icon, with no text and no `rel="next"`, where the direction lives only in the `aria-label` (e.g. "Go to next search page"), are now picked up and followed. This fixes eBay search results.
- **DuckDuckGo's HTML site** — its Previous/Next are `<input type="submit" value="Next">` buttons, and the label was being read from the (empty) element text instead of the `value`. Page Turner now reads the value, so those buttons work.

## Under the hood

- Added QUnit coverage for both new cases: icon-only anchors matched via `aria-label`, and submit inputs matched by their `value`.
- No new permissions requested — the manifest change is limited to the version and store description wording.
